# Restaurant Billing System (DineFlow)

This project is a high-performance, modern restaurant billing application built with a full-stack architecture. It serves as a comprehensive demonstration of the various technologies and methodologies I have mastered.

## 🛠️ Integrated Technologies & Tools

### Frontend
- **React.js**: Main UI framework for the single-page application.
- **Tailwind CSS**: Utility-first CSS framework for premium, responsive design.
- **Framer Motion**: Advanced animation library for smooth UI transitions and micro-interactions.
- **Lucide React**: Icon library for a clean and modern aesthetic.

### Backend
- **FastAPI / Python**: High-performance REST API for the core billing logic.
- **Python (Flask/Django)**: Alternative backend implementations provided in the `examples/` directory.

### Database
- **SQLite**: Default database for local development and simplicity.
- **MongoDB / MySQL**: Configuration and client examples for scalable database integration in `database_config.py`.

### DevOps & Methodologies
- **GitHub Actions**: Automated CI/CD pipeline for building and testing the application.
- **REST APIs**: Standardized communication between frontend and backend.
- **Version Control**: Full project lifecycle management using Git/GitHub.

## 📂 Project Structure
- `frontend/`: React + Vite application.
- `main.py`: Core FastAPI backend.
- `app.py`: Desktop-based Tkinter version of the billing software.
- `database_config.py`: Multi-database configuration (SQLite, MySQL, MongoDB).
- `.github/workflows/`: CI/CD automation.
- `examples/`: Implementation examples of alternative technologies (Flask, Django, etc.).
- `data/`: Database and generated PDF/CSV reports.
